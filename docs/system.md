# これは、このプロジェクトの根幹となるシステムや処理についての仕様を定義するMarkdownです。
このMarkdownでは、ゲームの基本的なシステムや処理の流れ、そしてそれらがどのように機能するかについて説明します。ゲームのシステムは、プレイヤーがゲームを楽しむための基盤となるものであり、ゲームのルールやメカニクスを定義する重要な要素です。

## ゲームの基本的なシステム
このゲームは、プレイヤーが音楽に合わせてノーツを入力するリズムゲームです。プレイヤーは、画面上に表示されるノーツをタイミングよく入力することでスコアを獲得し、コンボをつなげていきます。ゲームのシステムは、以下のような要素で構成されています。
- ノーツの種類と属性
- 判定方法とスコア計算
- コンボシステム
- HPシステム
- ゲームオーバー条件
- ゲームの進行と難易度設定
これらの要素は、プレイヤーがゲームを楽しむための基盤となるものであり、ゲームのルールやメカニクスを定義する重要な要素です。次のセクションでは、これらの要素について詳しく説明します。

### ノーツの種類と属性
docs/judge.mdやdocs/charts.mdで詳しく説明されているように、ノーツには様々な種類と属性があります。これらは、プレイヤーの入力猶予や、スコア計算、コンボの増加などに影響を与える重要な要素です。ノーツの種類や属性は、ゲームの難易度やプレイスタイルに大きな影響を与えます。

### 判定方法とスコア計算
docs/judge.mdで説明されているように、ノーツの判定方法やスコア計算は、プレイヤーの入力タイミングやノーツの種類、属性によって異なります。正確な入力が求められるノーツや、特定の条件を満たすことで高得点が獲得できるノーツなど、様々な要素が組み合わさっています。

### コンボシステム
- プレイヤーが連続してノーツを正確に入力することで、コンボが増加します。コンボが増えることによる直接の効果はありません。
- 特殊演出は、難易度設定に関係なく、条件を満たしかつ、すべてのノーツを遊び終えた際に発生します。
#### 特殊演出
##### ダメージ系
- **ゲームオーバー**は、プレイヤーのHPが0になった際に発生します。HPは、Damageノーツに触れることで減少し、0になると、ゲームが演奏中でも即刻終了し、ゲームオーバー演出が発生します。
- **FULL HEALTH**は、すべてのノーツを遊び終えた際、HPが満タンの状態であった場合に、FULL HEALTH演出が発生します。

- **NO DAMAGE**は、すべてのノーツを遊び終えた際、Damage判定が一度も発生しなかった(ダメージノーツに触れなかった)場合に、NO DAMAGE演出が発生します。

```
[FULL HEALTH]と、[NO DAMAGE]は、重複してリザルトに表示できます。
```

##### コンボ系
- **フルコンボ**はすべてのノーツを遊び終えた際、すべてのノーツでGood以上の判定を獲得した場合に、フルコンボ演出が発生します。なお、Damageノーツはコンボに影響を与えません

- **ALL PERFECT**はすべてのノーツを遊び終えた際、すべてのノーツでPerfect以上の判定を獲得した場合に、ALL PERFECT演出が発生します。なお、Damageノーツは影響を与えません

- **ALL CRITICAL**はすべてのノーツを遊び終えた際、すべてのノーツでPERFECT以上の判定を獲得し、かつ、クリティカルの追加加点が存在するノーツをすべてC-PERFECT以上を獲得した場合に、ALL CRITICAL演出が発生します。なお、Damageノーツは影響を与えません

```
重複してリザルトに表示できるコンボ系演出はありません。
ただし、ダメージ系特殊演出とコンボ系特殊演出は重複表示することができます。
```

### HPシステム
- プレイヤーは、ゲーム開始時に最大HPを持っています。HPは、Damageノーツに触れることで減少し、0になると、ゲームが演奏中でも即刻終了し、ゲームオーバー演出が発生します。
- HPは、judge.mdに記載されている通りに増減します。HPの増減は、ゲームの難易度やプレイスタイルに大きな影響を与えます。

### ゲームオーバー条件
- プレイヤーのHPが0になった際に、ゲームオーバー演出が発生します。

### ゲームの進行と難易度設定
- ゲームは、プレイヤーが選択した楽曲と難易度に応じて進行します。
- ゲームに表示される難易度は、楽曲ごとに複数用意されており、プレイヤーは自分のスキルや好みに応じて選択できます。

## 内部処理について
ゲームの内部処理は、プレイヤーの入力やノーツの生成、スコア計算など、ゲームの基本的なシステムを実現するためのコードやアルゴリズムで構成されています。これらの処理は、ゲームのパフォーマンスやプレイヤーの体験に大きな影響を与えるため、効率的かつ正確に実装される必要があります。

### ノーツの生成と管理
- データは、Charts/(曲ID)/(難易度).usc という楽曲データに基づいて生成されます。
- uscファイルは、UTF-8でエンコーディングされたjson形式のファイルであり、ノーツの種類や属性、タイミングなどの情報を含んでおり、ゲームはこれらの情報をもとにノーツを生成し、管理します。

### プレイヤーの入力処理
- プレイヤーの入力は、ゲームがリアルタイムで処理し、ノーツの判定やスコア計算に反映されます。
- 入力処理は、プレイヤーの入力タイミングやノーツの種類、属性に基づいて行われ、正確な入力が求められるノーツや、特定の条件を満たすことで高得点が獲得できるノーツなど、様々な要素が組み合わさっています。
- ノーツの幅が3レーンなど、太い場合は、そのノーツが覆っているレーンのキーすべてを用いて入力することができます。
- 例えば、幅が3レーンのノーツがレーン2からレーン4にかかっている場合、プレイヤーは、レーン2、レーン3、レーン4のいずれかを入力することで、そのノーツを処理することができます。

#### 使用するキー
- "asdfghjkl;:]"というJISキーボードのAから右のキーが、ノーツ入力に使用されます。
- これらのキーは、12個あるレーンに対応しており、プレイヤーはこれらのキーを使用してノーツを入力します。
- なお、ゲーム内では、これらのキーは、レーン1からレーン12に対応しています。例えば、"a"キーはレーン1に対応し、"s"キーはレーン2に対応します。

### スコア計算
judge.mdで説明されているように、スコア計算は、プレイヤーの入力タイミングやノーツの種類、属性によって異なります。正確な入力が求められるノーツや、特定の条件を満たすことで高得点が獲得できるノーツなど、様々な要素が組み合わさっています。

### ゲームの進行管理
- ゲームは、プレイヤーが楽曲を選択し、難易度を選択することで開始されます。

- なお、開始前にはレーンスピードと判定調整(オプション)を選択することができます。


#### レーンスピード
- レーンスピードは、ノーツが流れてくる速度を調整するオプションであり、選択肢0.1刻み(10px刻み)。プレイヤーは自分の好みに応じて選択できます。
- 例えば、レーンスピードを1.0に設定した場合、ノーツは通常の速度で流れてきますが、レーンスピードを2.0に設定した場合、ノーツは通常の2倍の速度で流れてきます。逆に、レーンスピードを0.5に設定した場合、ノーツは通常の半分の速度で流れてきます。
- 通常の速度はレーンスピード1.0であり、画面上に表示されるノーツは、1秒間に100pxの速度で流れてきます。レーンスピードを変更することで、ノーツの流れる速度を調整することができます。

#### 判定調整
- 判定調整では、判定調整Aと判定調整Bの2種類のオプションが用意されており、選択肢0.1刻み(0.16ms単位)で調整できます。プレイヤーは自分の好みに応じて選択できます。それぞれ(+5.0~-5.0で調整可能)
- 判定調整Aでは、ノーツのタイミングと音楽のタイミングをずらすオフセットをつけるという調整方法です。
- 判定調整Bでは、入力の+-の基準を0msからずらすという調整方法です。
- 判定調整Aと判定調整Bは、同時に選択することができます。例えば、判定調整Aで+1.0のオフセットをつけ、判定調整Bで-0.5の入力タイミングの基準をずらすことができます。
- なお、この判定調整のuiでは、選択したオプションと調整値をわかりやすく表示する必要があります。例えば、判定調整Aで+1.0のオフセットをつけ、判定調整Bで-0.5の入力タイミングの基準をずらす場合は、「判定調整A: +1.0, 判定調整B: -0.5」といった形で表示する必要があります。

#### オプション選択
- プレイヤーは、ゲーム開始前に、レーンスピードと判定調整のオプションを選択することができます。これらのオプションは、プレイヤーの好みやプレイスタイルに応じて、ゲームの難易度や体験をカスタマイズするためのものです。
- 判定調整は+-ボタンを用いて、0.1刻みで選択できます。
- レーンスピードは、同じく+-ボタンで0.1刻みで選択できます。
    - レーンスピードのでは、++と--ボタンもあり、これを押すと、現在選択しているレーンスピードから1.0刻みで上げ下げすることができます。

- ゲームの進行は、プレイヤーの入力やノーツの生成、スコア計算などの処理によって管理されます。


- 難易度を選択した後、ゲームは、選択された楽曲と難易度に応じてノーツを生成し、プレイヤーの入力を処理し、スコアを計算します。
- 選択した直後からノーツが来るとプレイヤーの体験を損ねるため、選択してから2秒程度で"準備はOK?"と表示し、その後、準備はOKの表示を消してから3秒程度の猶予時間を設けたのち、譜面の再生を開始する必要があります。
- なお、この「準備はOK?」の表示は、選択した楽曲と難易度に応じて、ゲームが開始される前の準備段階で表示されるものであり、プレイヤーがゲームを開始するための合図となります。
- プレイヤーは、「準備はOK?」の表示が消えた後、ノーツが画面上に表示されるのを待ち、ノーツが判定ラインに重なったタイミングで入力する必要があります。

- ノーツは上から下に流れてくるため、プレイヤーは、ノーツが判定ラインに重なったタイミングで入力する必要があります。
- ゲームが開始されると、プレイヤーは、画面上に表示されるノーツを判定ラインに重なったタイミングで入力することでスコアを獲得し、コンボをつなげていきます。
