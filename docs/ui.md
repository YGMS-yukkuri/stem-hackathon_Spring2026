# これは、このプロジェクトのUIについて説明・定義するMarkdownです。
このMarkdownでは、ゲームのユーザーインターフェース（UI）について説明します。UIは、プレイヤーがゲームとどのように対話するかを定義する重要な要素であり、ゲームの操作性やプレイヤーの体験に大きな影響を与えます。

## ゲームのUI要素
このゲームのUIは、プレイヤーがゲームを楽しむための情報や操作を提供するための要素で構成されています。以下は、ゲームの主要なUI要素の例です。
- **スコア表示**: プレイヤーの現在のスコアを表示
- **コンボ表示**: プレイヤーの現在のコンボ数を表示
- **HPバー**: プレイヤーの現在のHPを視覚的に表示
- **ノーツ表示**: 画面上に表示されるノーツを視覚的に表現
- **判定エフェクト**: プレイヤーの入力に対する判定結果を視覚的に表現
- **リザルト画面**: ゲーム終了後にプレイヤーの成績を表示する画面
これらのUI要素は、プレイヤーがゲームを楽しむための情報や操作を提供するための重要な役割を果たします。次のセクションでは、これらのUI要素について詳しく説明します。

### スコア表示
- プレイヤーの現在のスコアをリアルタイムで表示します。スコアは、プレイヤーの入力に対する判定結果やノーツの種類、属性などによって変動します。
- カンマ区切りで表示されます。例えば、スコアが1000の場合は「1,000」と表示されます。
- スコア表示は、画面の上部中央、コンボの左隣に配置されます。

### コンボ表示
- プレイヤーの現在のコンボ数をリアルタイムで表示します。コンボは、プレイヤーが連続してノーツを正確に入力することで増加します。
- カンマ区切りで表示されます。例えば、1000コンボの場合は「1,000」と表示されます。
- コンボ表示は、画面の上部中央に配置されます。スコア表示の右隣に配置されます。

### HPバー
- プレイヤーの現在のHPを視覚的に表示します。HPは、Damageノーツに触れることで減少し、0になるとゲームオーバーになります。
- スコア表示とコンボ表示の下に配置されます。HPバーは、プレイヤーが現在のHPを一目で確認できるように、視覚的にわかりやすいデザインで表示されます。
- HPバーは、プレイヤーのHPの割合に応じて、色やデザインが変化します。
    - HPが100%の場合は緑色、HPが50%以下の場合は黄色、HPが25%以下の場合は赤色というように、プレイヤーのHPの状態を視覚的に表現します。
- HPの値は数値で表示せず、HPバーの長さや色の変化で表現します。

### ノーツ表示
- ノーツは画面上部の奥から手前に向かって流れるように表示されます。プレイヤーは、ノーツが判定ラインに到達したタイミングで入力を行います。(判定ラインは、画面の下部に配置されます)
    - 画面の上部(HPやスコアの表示の下)から、ノーツが流れてきます。判定ラインは、画面の高さ10%の位置に配置されます。
- 画面上に表示されるノーツを視覚的に表現します。ノーツの種類や属性に応じて、異なるデザインやエフェクトが適用されます。
- ノーツは、画面の上部から下部に向かって流れるように表示されます。プレイヤーは、ノーツが判定ラインに到達したタイミングで入力を行います。

- ノーツは、最も中央寄り(0に近い)から外側(絶対値が大きくなる方向)に向けてサイズを適用します。
    - 左端は-5.5,右端は5.5からスタートし、例えば、描画したいノーツの幅が3レーンで、レーン2からレーン4にかかっている場合、譜面ファイルに記載されているレーンはレーン2に位置し、サイズは1.5になります。

#### ノーツデザイン
- ノーツのデザインは、ノーツの種類や属性に応じて異なります。
    - Singleノーツ
        - オレンジ色で角が丸まった長方形で表示されます。
    - Single-Criticalノーツ
        - 金色で角が丸まった長方形で表示されます。
    - Single-Traceノーツ
        - 水色で角が丸まった長方形で表示されます。
    - Single-Directionノーツ
        - ピンク色で角が丸まった長方形で表示され、ノーツの中央に上向き三角が付属しています
    - Damageノーツ
        - 赤色で横長の楕円形で表示されます。
    - Slideノーツ
        - 青色で角が丸まった長方形で表示されます。
        - 始点から中継点・終点までは場所や幅をeaseオプションに応じて曲線で描画します。
    - Guideノーツ
        - 半透明の緑色の線で表示されます。線の太さはsizeで指定されます。
        - 始点から中継点・終点までは場所や幅をeaseオプションに応じて曲線で描画します。
### 判定エフェクト
- プレイヤーの入力に対する判定結果を視覚的に表現します。例えば、Perfect、Great、Good、Badなどの判定に応じて異なるエフェクトが表示されます。
- エフェクトのほか、各ノーツの中央でかつ、判定したノーツから少し上に判定結果が表示されます。
- 判定結果の文字は、ノーツによって色などが変わります。例えば、C-Perfectは金色、Perfectは黄色、Greatはオレンジ色、Goodは緑色、Badは黒色などのように、判定結果の文字のデザインが変わります。
- Damageノーツに対する判定結果は、SAFE、DAMAGEであり、SAFEは金色、DAMAGEは赤色の文字で表示されます。

### リザルト画面
- ゲーム終了後にプレイヤーの成績を表示する画面です。スコア、コンボ数、判定結果などの情報を一覧表示します。
- 特殊演出の表示も行います。例えば、FULL HEALTH、NO DAMAGE、FULL COMBO、ALL PERFECT、ALL CRITICALなどの特殊演出が発生した場合、それらをリザルト画面に表示します。
- リザルト画面は、ゲームの終了後に自動的に表示されます。プレイヤーは、リザルト画面で自分の成績を確認し、次のゲームに進むことができます。
- リザルト画面には、次のゲームに進むためのボタンや、タイトル画面に戻るためのボタンなども配置されます。

## UIのデザインとユーザビリティ
ゲームのUIは、プレイヤーがゲームを楽しむための情報や操作を提供するための重要な要素であり、デザインとユーザビリティが非常に重要です。UIのデザインは、ゲームのテーマやスタイルに合わせて、視覚的に魅力的でわかりやすいものにする必要があります。また、ユーザビリティは、プレイヤーが直感的に操作できるようにするための要素であり、UIの配置や操作方法などを工夫する必要があります。例えば、スコア表示やコンボ表示は、プレイヤーが常に確認できる位置に配置し、HPバーは視覚的にわかりやすいデザインで表示するなど、プレイヤーがゲームを楽しむための情報を適切に提供することが重要です。

## 内部処理について
ゲームのUIの内部処理は、プレイヤーの入力やゲームの状態に応じて、UI要素を更新するためのコードやアルゴリズムで構成されています。これらの処理は、ゲームのパフォーマンスやプレイヤーの体験に大きな影響を与えるため、効率的かつ正確に実装される必要があります。

### UIの更新と管理
- プレイヤーの入力やゲームの状態に応じて、UI要素をリアルタイムで更新するためのコードやアルゴリズムが必要です。例えば、プレイヤーがノーツを入力した際に、スコア表示やコンボ表示、HPバーなどのUI要素を適切に更新するための処理が必要です。
- UI要素の管理は、ゲームの状態やプレイヤーの入力に応じて、UI要素を適切に表示・非表示にするためのコードやアルゴリズムが必要です。例えば、ゲームオーバー時にはHPバーを非表示にする前にゲームオーバーを明示的にユーザーに示したあと、数秒おいてからリザルト画面を表示するなどの処理が必要です。

- UIの更新と管理は、ゲームのパフォーマンスやプレイヤーの体験に大きな影響を与えるため、効率的かつ正確に実装される必要があります。例えば、UI要素の探索は、初回ロード時にまとめて行いゲーム中に参照を行わないようにするなど、パフォーマンスを考慮した実装が必要です。

# UIのアニメーションとエフェクト
- UIのアニメーションやエフェクトは、プレイヤーの入力やゲームの状態に応じて、UI要素を視覚的に魅力的にするためのコードやアルゴリズムで構成されています。例えば、スコア表示やコンボ表示が更新される際に、アニメーションやエフェクトを適用することで、プレイヤーの入力に対するフィードバックを強化し、ゲームの楽しさを向上させることができます。
- UIのアニメーションやエフェクトは、ゲームのテーマやスタイルに合わせて、視覚的に魅力的なものにする必要があります。例えば、スコア表示が更新される際に、スコアが増加するエフェクトや、コンボ表示が更新される際に、コンボ数が増加するエフェクトなどを適用することで、プレイヤーの入力に対するフィードバックを強化し、ゲームの楽しさを向上させることができます。
- UIのアニメーションやエフェクトは、ゲームのパフォーマンスやプレイヤーの体験に大きな影響を与えるため、効率的かつ正確に実装される必要があります。例えば、アニメーションやエフェクトの処理は、ゲームのフレームレートを維持するために、適切なタイミングで実行されるようにするなど、パフォーマンスを考慮した実装が必要です。

- コンボの更新のアニメーションは、コンボ数が増加する際に、コンボ表示が拡大してから元のサイズに戻るようなエフェクトを適用することで、プレイヤーの入力に対するフィードバックを強化し、ゲームの楽しさを向上させることができます。

- 判定も、少し上に出す際に目標位置からすこしだけ下から出現させ、減速アニメーションで目標位置に到達するようなエフェクトを適用することで、プレイヤーの入力に対するフィードバックを強化し、ゲームの楽しさを向上させることができます。
- なお、1つのノーツに対して、複数の判定が発生することはないため、判定エフェクトも1つのノーツに対して1つのみ表示されるようにします。
